# Knowledge Archive

AI 시대 개발자의 가치는 두 가지라고 생각합니다. **일을 만들어서 AI에게 시키는 것**, 그리고 **AI가 만든 결과물을 잘 리뷰하는 것**.

이 저장소는 후자를 위한 것입니다.

AI가 작성한 코드를 리뷰하려면 전문성이 필요합니다. 이를 위해 공식 문서를 기반으로 학습하고 있으며, 영어 원문을 기록하고 기술 면접 형식으로 복습합니다.

---

## 어떻게 쌓나요?

공식 문서를 읽고 이해한 뒤에도, 시간이 지나면 기억나지 않습니다. 다시 찾으려면 어디서 봤는지조차 떠올리기 어렵습니다.

이 저장소는 공식 문서를 읽는 시점에 Q&A 형태로 바로 축적합니다. 이때 사실을 잘 저장하는 데 집중합니다 — 원문과 해석을 분리하고, 출처를 반드시 명시하고, 태그·파일명·폴더 경로 세 가지로 검색할 수 있게 구조화합니다. 사실이 이렇게 정확하고 찾기 쉬운 형태로 쌓이면, 같은 데이터를 학습·복습·팩트체크·면접 준비 등 여러 용도로 활용할 수 있습니다.

### /digest — 공식 문서를 읽으며 실시간으로 기록

공식 문서 URL을 입력하면, 단락 단위로 해설하면서 기록할 만한 Q&A를 제안합니다.

> - 영어 원문을 한 문장씩 해설하고, 3줄 요약을 제공합니다
> - 기존 기록과 중복되는지 자동으로 확인합니다
> - 기록할 가치가 있는 내용이면 질문을 제안하고, 승인하면 즉시 저장됩니다

```
/digest ON https://react.dev/reference/react/useMemo
```

> 자세한 내용은 [SKILL.md](.claude/skills/digest/SKILL.md)를 참고해 주시면 감사하겠습니다.

## 어떻게 복습하나요?

기록만 하고 다시 보지 않으면 의미가 없습니다. 혼자 복습하면 아는 것만 반복하게 되고, 모르는 부분을 스스로 발견하기는 어렵습니다.

### /review — 시니어 면접관이 질문합니다

축적된 Q&A를 기반으로 시니어 개발자가 기술 면접을 진행하는 것처럼 질문합니다. 답을 알려주지 않고, 빠진 부분은 꼬리 질문으로 유도합니다.

> - 한 번에 하나의 질문만 던지고, 충분히 시도한 뒤에 정답을 공개합니다
> - 답에서 빠진 부분이 있으면 직접 알려주지 않고, 꼬리 질문으로 스스로 찾게 합니다
> - 끝나면 틀린 답변 요약, 개선 방향, 심화 주제를 정리한 리포트를 제공합니다

```
/review 네트워크 관련 질문으로 복습하자
```

> 자세한 내용은 [SKILL.md](.claude/skills/review/SKILL.md)를 참고해 주시면 감사하겠습니다.
