# Answers

## [UNVERIFIED] 기억장치란 무엇인가?

### User Answer
데이터를 저장하기 위한 장치다.
데이터를 영구적으로 또는 임시적으로 저장하기 위해 필요하다.

---

## [UNVERIFIED] 주기억장치란 무엇인가?

### User Answer
데이터를 임시적으로 저장할 수 있는 장치이며, 전원 공급이 중단되면 저장되어 있던 모든 데이터가 날아가는 특징이 있다.
보조기억장치보다 읽기/쓰기 속도가 더 빠르며, 같은 용량 대비 더 비싸다.
같은 말로 Main Memory, RAM이 있다.

---

## [UNVERIFIED] 보조기억장치란 무엇인가?

### User Answer
데이터를 영구적으로 저장할 수 있는 장치이며, 전원 공급이 중단되어도 데이터가 날아가지 않는다.
일반적으로 주기억장치보다 읽기/쓰기가 느린 대신 같은 용량 대비 더 저렴하다.
같은 말로 2차 기억장치가 있다.

> #### User Annotation:
> 영어로는 Storage라는 표현을 쓴다.
> Memory라는 표현을 쓰면 주기억장치와 혼동할 수 있어 잘 쓰지 않는다.

## [UNVERIFIED] 기억장치의 가격, 용량, 속도는 어떤 관계인가?

### User Answer
CPU는 주기억장치에서 데이터를 가져올 때, 해당 데이터가 주기억장치에 없으면 보조기억장치에서 주기억장치로 이동시킨 후에 가져와야 한다.
(CPU는 보조기억장치에서 곧바로 읽을 수 없다.)

주기억장치의 용량이 작으면 이런 일이 더 빈번하게 발생하고, 그때마다 CPU는 오래 기다릴 수밖에 없다.
그렇다고 메모리 용량을 키우면 비용이 많이 드는 문제가 있다.
용량도 크면서 읽기/쓰기가 빠른 기억장치는 가격이 비싸기 때문이다.

그래서 기억장치를 용도별로 다양하게 계층적으로 구성하게 되었다.

- 1티어: CPU 안에 있는 레지스터 (데이터 버스도 CPU 내부 전용을 따로 쓸 정도로 매우 빠르다)
- 2티어: 캐시메모리
- 3티어: 주기억장치

티어가 높을수록 CPU의 액세스 빈도가 잦고, 비싸고, 용량이 작고, 빠르다.

프로그램 실행 중에 CPU 내부 레지스터에 없는 데이터가 주기억장치에 있는 경우, 이걸 CPU 내부로 가져오기 위해서는 (비교적) 상당한 시간이 필요하다.
이런 문제를 해결하기 위해 중간에 캐시메모리를 둔다.
CPU와 주기억장치 간의 속도 차이를 보완하기 위한 버퍼 같은 역할이다.

## [UNVERIFIED] RAM이란 무엇인가?

### User Answer
CPU가 전송한 주소에 의해서 Access된다.
데이터를 읽고 쓸 수 있으며, 휘발성이다.
칩 내에서 어느 위치에 데이터가 있더라도 Access 시간이 동일하다.

## [UNVERIFIED] ROM이란 무엇이며, 주로 어떤 내용이 저장되는가?

### User Answer
Read Only Memory의 약자로, 뜻 그대로 읽기만 가능한 기억장치이다.
비휘발성이다.

주로 저장되는 내용:
- 시스템 초기화 프로그램, 진단 프로그램
- 자주 사용되는 함수들과 서브루틴들
- CU(Control Unit)의 마이크로 프로그램
