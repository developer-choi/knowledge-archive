# Digest

공식문서를 학습하고, 중요한 내용을 Q&A로 knowledge/ 문서에 저장한다.

## 공통 규칙

모든 모드에서 Q&A를 저장할 때 적용되는 규칙이다.

### 내용 분류
- 영어 원문이 있으면 그대로 보존하여 Official Answer로 사용
- 한글 메모는 Annotation으로 분류
- **영어 원문 없이 한글만 제공된 경우**: User Answer로 작성. AI가 영어 Official Answer를 생성하지 않는다

### 이미지 alt text 필터링
사용자가 붙여넣은 텍스트에는 이미지의 alt text가 섞여 들어올 수 있다. 주어-서술어 구조가 없는 나열형 문장, 이미지 캡션으로 추정되는 문장은 Official Answer에서 제외한다. 해설 시에도 본문과 구분하여 취급한다.

### 범위 제한
사용자가 제공한 내용의 범위 내에서만 Q&A를 작성한다. 출처 URL을 방문하여 사용자가 언급하지 않은 추가 내용을 가져오지 않는다.

### TODO 매칭
Q&A를 새로 만들기 전에, 기존 knowledge/ 문서에 사용자 내용과 관련된 `[TODO]` 질문이 있는지 검색한다.
- **매칭됨**: 해당 `[TODO]` 질문의 Answer 섹션에 내용을 채우고, Questions 목록과 Answers 제목 양쪽에서 `[TODO]` 접두사를 제거
- **매칭 안 됨**: 새 Q&A를 작성

TODO 답변을 채울 때, 해당 질문에 대한 **직접적인 답**이 되는 내용을 작성한다. 같은 파일 내 다른 답변의 문장을 그대로 복사하지 않는다.

## 모드 1: 대화형 학습 (ON/OFF)

### ON — 모드 시작

[production-guide.md](../../contexts/production-guide.md)의 **Before** 실행.

출처 URL을 기록하고 아래 안내를 출력한다:

```
digest 모드 시작.
단락을 붙여넣으면 해설합니다.
기억할 만한 내용은 질문으로 만들어 제안합니다.
승인하면 저장 목록에 추가, /digest OFF 하면 파일로 생성합니다.
단락을 붙여넣어 주세요.
```

### 루프 — 단락 처리

사용자가 단락을 붙여넣을 때마다:

#### 1. 해설

문장 단위로 한글 해설한다. 개발자가 실무에서 이해할 수 있도록 구체적으로 설명한다.

- 영어 원문을 인용하고, 바로 아래에 한글로 풀어 설명
- 추상적 개념은 **코드 예시**로 보여준다
- 추상적 개념은 개발자가 아는 실무 상황이나 일상에 **비유**해서 설명한다
- "왜 이렇게 동작하는가", "이게 없으면 어떻게 되는가"까지 설명
- 흔한 오해가 있는 문장은 "주의: ~라고 오해하기 쉽지만 실제로는 ~" 형태로 짚어준다

#### 1-1. 요약

해설이 끝나면 단락 전체의 핵심을 비유와 함께 3줄 이내로 정리한다.

#### 2. 중복 검사

해설 후, 기억할 가치가 있는 내용이면 질문을 만들기 **전에** 기존 knowledge/ 문서에 동일한 내용이 있는지 확인한다.

- 태그, 파일명, 폴더 기반으로 검색
- 이미 있으면 사용자에게 알리고 건너뛴다

#### 3. 질문 제안

중복이 아닌 경우, 질문을 제안한다. 질문 작성 시 아래 원칙을 따른다:

- **면접 질문처럼**: 맥락 없이 읽어도 무슨 말인지 이해되어야 한다. 전문 용어로 냅다 시작하지 말고, 상황/배경을 질문 안에 포함한다.
- **문제→해결 구조**: "이런 문제가 있다. 어떻게 해결하는가?" 형태가 recall에 좋다.
- **리스트 암기 금지**: "3가지는?" 식의 나열 질문 대신, 왜 그런지를 묻는 질문으로 만든다.

질문 유형:
- 일반 질문: "X란 무엇인가?"
- 오개념 타파: "X는 Y를 하는가?" (답이 아니오인 질문)
- 실무 연결: "X가 실제로 해결하는 문제는?"
- 문제→해결: "이런 문제가 있었다. 어떻게 해결하는가?"

제안할 때 질문과 함께 **Official Answer(영어 원문 그대로) + 한글 해설**을 같이 보여준다.
사용자가 승인하면 저장 목록에 추가, 거부하면 버린다.

단락마다 무조건 질문을 만들려고 하지 않는다. 사용자가 실무에서 써먹을 수 있는 내용인지, 기존 상식과 다른 내용인지를 따진다. 가치가 없다고 판단하면 사유와 함께 "이건 ~한 이유로 학습 안 해도 될 것 같은데 넘어갈까요?"로 사용자에게 확인받는다. 최종 판단은 사용자가 한다.

### OFF — 모드 종료

저장 목록에 승인된 질문이 있으면 [production-guide.md](../../contexts/production-guide.md)의 **After** 실행.

승인된 질문이 없으면 파일을 생성하지 않는다.

## 모드 2: 즉시 저장

사용자가 학습 내용을 "필기해줘", "기록해줘" 등과 함께 제공하면, 해설 없이 바로 Q&A를 만들어 저장한다.

### 작업 순서

1. [production-guide.md](../../contexts/production-guide.md)의 **Before** 읽기
2. 내용 파악: 공통 규칙(내용 분류, 범위 제한) 적용
3. TODO 매칭: 공통 규칙 적용
4. Q&A 작성: [content-format.md](../../contexts/content-format.md)에 따라 작성
5. [production-guide.md](../../contexts/production-guide.md)의 **After** 실행
