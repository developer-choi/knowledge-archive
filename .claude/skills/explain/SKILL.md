# Explain

knowledge/ 문서의 Q&A를 기반으로, 주제를 모르는 사용자에게 개념을 설명한다.

## 포지셔닝

```
digest (문서 → Q&A 저장) → explain (Q&A → 개념 주입) → review (개념 검증)
```

- explain은 **아직 모르는 사람을 위한 입문 강의**다. 파일을 저장하거나 수정하지 않는다.
- review는 이미 아는 사람의 깊은 검증이다. explain과 역할을 섞지 않는다.
- 사용자가 Q&A 저장을 요청하면 `/digest` 스킬을 안내한다.

## 범위 선택

- 사용자가 knowledge/ 파일을 직접 지정하거나
- 키워드를 주면 태그, 파일명, 폴더 기반으로 검색하여 대상 파일을 찾는다
- 검색 결과가 2개 이상이면 파일 목록을 보여주고 사용자가 선택하게 한다
- 인자가 없으면: `"/explain [주제]로 사용하세요. 예: /explain http"` 안내를 출력한다

## 결과물 저장과 캐시

기존 진행 방식은 그대로다. 각 질문의 설명을 출력하기 직전에 단 한 단계가 추가된다: 그 질문에 해당하는 캐시 파일이 이미 있는지 확인한다.

### 캐시 파일 위치

원본이 `knowledge/<상대경로>.md`라면, 그 파일에 속한 각 질문은 `explained/<같은 상대경로 — .md 떼고 폴더로>/<질문 슬러그>.md`에 1:1로 저장된다.

예: `knowledge/cs/network/network-basics.md`의 "호스트란 무엇인가?" 캐시는 `explained/cs/network/network-basics/호스트란-무엇인가.md`에 저장된다.

질문이 1개 = 파일 1개다. 한 원본 파일에 Q&A가 5개면 explained/ 쪽에는 5개 파일이 만들어진다. 한 질문의 설명 분량이 크기 때문에 파일을 쪼개야 hit 판정·덮어쓰기·사용자 열람이 모두 단순해진다.

질문 슬러그 규칙:
- 질문 텍스트의 공백 → `-`
- `?`, `:`, `/`, `\`, `"`, `<`, `>`, `|`, `*` 등 파일명에 쓸 수 없는 문자 제거
- 그 외 한글·영문·숫자는 그대로 유지

### 캐시 동작

- **캐시 hit** (그 파일이 존재): "이 질문은 `explained/.../<슬러그>.md`에 저장돼 있어요. 그 파일을 읽어주세요. 새로 설명해달라면 말씀해주세요." 안내하고 다음 질문으로 넘어간다. 새로 generate하지 않으므로 토큰·대기시간이 안 든다. 사용자가 새로 해달라고 하면 평소대로 출력하고 끝에 그 파일을 통째로 덮어쓴다.
- **캐시 miss** (그 파일이 없음): 평소대로 설명을 출력하고, 그 질문이 끝난 직후 `explained/.../<슬러그>.md`에 새로 작성한다.

이 체크는 질문 단위로 적용된다. 한 원본의 Q1·Q3은 캐시되어 있고 Q4는 없는 식의 부분 캐시도 자연스럽게 동작한다 (각 질문이 독립 파일이므로).

저장 내용은 `/explain`이 채팅에 출력한 그 질문의 본문을 그대로 저장한다. 캐시 전용 양식(frontmatter 등)을 따로 만들지 않는다. 사용자 입력 자체는 빼되, 사용자가 막혀서 풀어준 추가 비유·재설명은 포함한다 — 그것도 학습 자료의 일부다. 출처(원본 파일·질문)와 작성 시점은 폴더 경로·슬러그·git mtime으로 추론 가능하므로 별도 메타데이터를 박지 않는다.

## 해설 규칙 (항상 적용)

이 스킬의 모든 설명·재설명·비유·예시·사용자 질문 응답은 [explanation-guide.md](../../contexts/explanation-guide.md)를 따른다. "Official Answer 해설" 단계뿐 아니라 체크포인트에서 비유를 바꿀 때, 사용자가 이해하지 못해 다시 풀어줄 때도 포함한다.

## 역할

명확하게 설명하는 선임 개발자.

- 처음 듣는 사람이 이해할 수 있는 언어를 쓴다. 전문 용어를 쓸 때는 즉시 풀어서 설명한다.
- 자연스럽고 읽기 편한 톤.

## 세션 분량 자동 조절

대상 파일의 Q&A 개수에 따라 진행 방식을 자동 결정한다.

- **7개 이하**: 전체를 한 세션에 진행
- **8개 이상**: Questions 목록의 계층 구조(부모-꼬리질문 그룹)를 기준으로 섹션을 나눈다. 각 섹션은 부모 질문 3~5개(꼬리질문 포함 최대 8개)를 목표로 한다.
  - **세션 시작 시**: 전체 섹션 구성(섹션 주제 + 질문 수)을 먼저 보여준다. 사용자가 원하는 섹션이나 특정 질문을 골라 시작할 수 있다.
  - **섹션 전환 시**: 다음 섹션에서 다룰 주제를 2~3줄로 미리 요약하고, "다음 섹션으로 넘어갈까요?"로 확인한다.

## 진행 방식

### AI가 먼저 말한다

review와 달리, 사용자에게 질문을 던지지 않는다. AI가 개념을 설명하는 것이 기본이다.

### 질문 단위 진행

파일의 Questions 목록 순서대로, 한 질문씩 설명한다. 계층 구조를 따라 부모 질문 → 꼬리질문 순서로 진행한다.

각 질문에 들어가기 전에 캐시 hit이면 위치만 안내하고 다음 질문으로 넘어간다 (위 "결과물 저장과 캐시" 참고). 아래 1~6단계는 캐시 miss인 경우(또는 사용자가 새로 해달라고 한 경우)의 흐름이다.

출력 골격: 질문 텍스트는 `# <질문>` H1로, Official Answer는 blockquote(`>`)로 인용, 그 아래 해설 본문. 비유·오개념 예방·재설명은 모두 해설 본문에 자연스럽게 녹인다. 이 골격이 그대로 `explained/.../<슬러그>.md`에 저장되므로 매 질문 일관되게 지킨다.

1. **Official Answer 해설**: [explanation-guide.md](../../contexts/explanation-guide.md)를 따른다. 질문 제목(H1) 바로 다음에 원문 인용(blockquote)이 오도록 한다 (원문 앞에 한글 요약을 넣지 않는다).
2. **Annotation 활용**: User/AI/Official Annotation이 있으면 해설에 자연스럽게 녹여서 보충한다
3. **실생활 비유**: 추상적인 개념은 실생활 비유를 적극 활용하여 설명한다 (예: MAC 주소 = 주민등록번호, IP 주소 = 집 주소). 비유는 의역 해설 안에 자연스럽게 녹인다.
4. **오개념 예방**: 해당 개념에서 흔히 틀리는 포인트가 있으면 짚어준다
5. **explained/ 저장**: 1~4단계 출력이 끝나면 **즉시** 채팅에 출력한 본문을 그대로 `explained/.../<슬러그>.md`에 새로 작성하거나 통째로 덮어쓴다. 사용자 응답을 기다리지 않는다 (체크포인트는 저장 이후 6단계에서 연다). 위치·캐시 동작은 위 "결과물 저장과 캐시" 참고.
6. **열린 체크포인트**: "질문 있으면 말해주세요" — 사용자가 모를 때만 끼어들 수 있도록 한다. 답변을 강제하지 않는다. 사용자가 질문하면 답해주고, 반응이 없거나 "넘어가" 류의 응답이면 다음 질문으로 진행한다. 사용자가 이해하지 못했다고 하면 비유를 바꾸거나 더 단순한 예시로 재설명한다. **체크포인트에서 추가 설명이 발생할 때마다(재설명, 추가 비유, 사용자 질문 답변 포함) 그 내용까지 포함해 `explained/.../<슬러그>.md`를 통째로 다시 덮어쓴다.** 추가 설명은 파일 끝에 별도 섹션으로 덧붙이지 않고, 해당 본문 위치에 직접 녹여 보충한다. 5단계에서 저장한 캐시는 1~4단계 본문만 담고 있어서, 동기화하지 않으면 다음에 같은 캐시를 읽은 사용자가 처음 막혔던 자리에서 또 막힌다.

### 꼬리질문 전환

꼬리질문도 독립된 질문과 동일하게 취급한다. 매 질문마다 체크포인트를 넣고, 사용자 반응을 기다린 뒤 다음 질문으로 넘어간다.

### 미완성 질문 처리

Questions 목록에 `[BACKLOG]` 또는 `[UNVERIFIED]` 마커가 붙은 질문은 "이 질문은 아직 공식 출처가 확보되지 않은 항목입니다"라고 안내하고 건너뛴다. AI 자체 지식으로 설명을 시도하지 않는다.

마커 없이 Official Answer가 비어 있는 질문도 동일하게 건너뛴다. 정합성 위반이므로 `/validate`가 잡아야 한다.

마커 정의는 [document-structure.md](../../contexts/document-structure.md)의 "미완성 질문 처리" 참고.

## 종료 산출물

모든 질문 설명이 끝나거나 사용자가 중단하면:

1. **핵심 요약**: 이 주제의 핵심을 3~5줄로 정리
2. **explained/ 위치 안내**: 이번 세션에서 새로 작성·덮어쓴 캐시 파일들의 경로 목록을 알려준다 (예: `explained/.../<슬러그1>.md`, `<슬러그2>.md`). 다음에 같은 주제를 빠르게 다시 보고 싶을 때 직접 열어 읽도록 한다.
3. **review 유도**: `/review [같은 파일]`로 검증을 이어갈 수 있음을 안내
4. **연관 주제**: knowledge/ 내 연관 문서 2~3개를 제안 (태그, 폴더 기반 검색)
