# 문서 구조 규칙

문서를 작성하거나 편집할 때 지켜야 하는 구조적 규칙입니다.

---

## 목차(TOC)와 본문 순서 동기화

질문 목록(Table of Contents)과 본문(Answers)의 질문 순서는 1:1로 일치해야 한다.

- 새로운 질문을 본문의 중간에 삽입했다면, 상단 목차(Questions)에서도 동일한 위치에 항목을 삽입했는가?
- 본문의 질문 순서를 변경했다면, 목차의 순서도 동일하게 변경했는가?
- 질문의 들여쓰기(계층 구조)가 목차와 본문 간에 논리적으로 일치하는가?
- 파일을 저장하기 전에 위에서 아래로 훑어보며 목차의 순서대로 본문이 전개되는지 검증했는가?

---

## 계층 구조 표현

### Questions 섹션
- **소제목(Header)**: 질문들이 카테고리로 분류되어 있다면, 해당 소제목을 유지합니다.
- **위계(Indent)**: 질문 하위에 꼬리 질문(들여쓰기, 하위 불렛)이 있다면, 이를 반영하여 중첩된 리스트 구조로 작성합니다.

예시:
```markdown
# Questions
- Main Question 1
  - Follow-up Question 1
  - Follow-up Question 2
- Main Question 2
```

### Answers 섹션
본문에서는 헤더 레벨(H2, H3 등)을 통해 위계를 표현할 수 있습니다.

예시:
```markdown
# Answers

## Main Question 1
...

## Follow-up Question 1
...

## Follow-up Question 2
...

## Main Question 2
...
```

## TODO 질문 처리

답변이 아직 작성되지 않은 질문은 다음과 같이 처리합니다:

1. **Questions 목록**: 질문 왼쪽에 `[TODO]` 접두사를 붙입니다.
   ```markdown
   - [TODO] 도메인이란 무엇인가?
   ```

2. **Answers 섹션**: 제목과 하위 구조(Keywords, Official Answer 등)를 미리 만들어두되 내용은 비워둡니다.
   ```markdown
   ## [TODO] 도메인이란 무엇인가?
   
   ### Official Answer

   ### Reference
   ```

## 허용 H1 헤딩

문서에서 사용할 수 있는 H1(`#`) 헤딩은 다음 3개로 제한한다.

| H1 | 용도 |
|---|---|
| `# Questions` | 질문 목차 |
| `# Answers` | 답변 본문 |
| `# TODO` | 문서 자체가 TODO |

이 외의 H1 헤딩(문서 제목 등)은 사용하지 않는다.

---

## 질문 추가/이동 규칙

- **문맥 고려**: 새로운 질문은 관련된 주제(Context) 근처에 배치합니다.
- **들여쓰기 활용**: 특정 질문에 종속된 심화 질문(꼬리 질문)이라면, 목차에서 들여쓰기를 사용하여 계층 구조를 명확히 합니다.

---

## 꼬리 질문이 다른 md에 있을 때

꼬리 질문이 너무 많아져서 별도 md 파일로 분리한 경우, 원본 파일 Questions 목차에 참조 링크로 유지한다. 이 경우 목차-본문 1:1 동기화 규칙의 예외로 인정한다 (답이 다른 파일에 있으므로).

형식: ``- [질문 → `파일명.md`](상대경로#앵커)``

- 같은 디렉토리·다른 디렉토리 구분 없이 동일 양식 사용
- 들여쓰기는 다른 꼬리 질문과 동일하게 유지
- 상대경로는 원본 파일 기준 실제 경로(같은 디렉토리면 파일명만, 다른 디렉토리면 `../` 등을 사용)

예시:
```markdown
- HTTPS 사이트 접속 시 브라우저가 HTML 데이터를 받기 전까지 거치는 네트워크 단계는?
  - [TLS negotiation에서 장기 키와 세션 키는 어떤 역할을 하는가? → `https.md`](../../cs/network/protocol/https.md#tls가-데이터를-암호화하는-과정에서-장기-키와-세션-키의-역할은)
```
