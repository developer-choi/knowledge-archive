# Web Storage에 PII 담지 마라

Storage(localStorage·sessionStorage)에 유저 개인정보(PII)는 담지 않는 것이 제일 좋다. 부득이하면 메모리에 두고, 그것도 안 되면 사용 즉시 삭제한다.

## 안전한 값 vs 위험한 값

같은 SNS 회원가입 플로우에서도 저장하는 값에 따라 위험도가 다르다.

**안전 (털려도 영향 작음)**:

- SNS 로그인 제공자 식별자 (예: `google`, `apple`, `kakao`)
- SNS 로그인 목적 (로그인 / 회원가입 / 재검증 / 채널연결)
- 리다이렉트 URL 경로

**위험 (PII)**:

- SNS 제공자가 준 사용자 ID (예: `sns_id`, 인가 코드)
- SDK가 전달한 사용자 정보 일체 (이름·이메일·전화 등)

## 세션 스토리지 함정

세션 스토리지는 탭이 닫히기 전까지 값을 유지한다. 회원가입 절차 시작 시점에 PII를 저장해두고 절차 끝까지 들고 가는 패턴은 PII가 길게 살아남게 만든다.

## 대안 우선순위

1. **메모리(컴포넌트 state · 전역 store)에 보관** — 새로고침으로 사라지므로 가장 안전
2. **부득이 Storage에 둘 경우 사용 즉시 삭제** — `get` → 사용 → 즉시 `remove`
