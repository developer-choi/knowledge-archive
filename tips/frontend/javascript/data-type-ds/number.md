# Number

## 나누기 — 분자/분모 0 검토 필요

```
1/0 = Infinity
-1/0 = -Infinity
0/0 = NaN
```

## NaN/Infinity는 어떠한 부등호연산 결과에도 false

```ts
/**
 * 안전하다의 기준은,
 * 1. NaN이 아니어야하고
 * 2. Infinity가 아니어야하며,
 * 3. SAFE_INTEGER 범위 안에 들어와야함.
 */
export function isSafeNumber(value: number): boolean {
  return !Number.isNaN(value) && Number.isFinite(value) &&
      Number.MIN_SAFE_INTEGER <= value && value <= Number.MAX_SAFE_INTEGER;
}
```

NaN/Infinity가 부등호에서 false 떨어지니, 이런 코드 작성 안 해도 되고,

```ts
return Number.MIN_SAFE_INTEGER <= value && value <= Number.MAX_SAFE_INTEGER;
```

딱 이거만 써도 된다.

이걸 좀 더 멋있게 표현하면

```ts
return Math.abs(value) <= Number.MAX_SAFE_INTEGER;
```

이렇게 표현이 가능하다.

## parseInt / parseFloat / Number 차이

**정수 (양수, 음수)**

| Input / Function | `parseInt` | `parseFloat` | `Number` |
|---|---|---|---|
| `"123456"` | 123456 | 123456 | 123456 |
| `"-123456"` | -123456 | -123456 | -123456 |
| `"+123456"` | 123456 | 123456 | 123456 |

**소수, 문자포함숫자, 공백포함숫자**

| Input / Function | `parseInt` | `parseFloat` | `Number` |
|---|---|---|---|
| `"123.456"` | 123 | 123.456 | 123.456 |
| `"123abc"` | 123 | 123 | NaN |
| `" 123456 "` | 123456 | 123456 | 123456 |

**결론 (기능)**

- `parseInt()`는 정수형으로 바꿔주고,
- `Number()`는 `parseFloat`과 제일 비슷하고,
- `Number` vs `parseXXX`는 숫자가 나오는 곳까지 파싱을 해준다는 차이가 있다.

**결론 (타입)**

- `parseXXX(str: string)`인데,
- `Number(val: any)`라는 차이가 있다.

```js
Number("123.0")
// 123
```

이것도 참고.

### 입력값이 빈 문자열

- `Number('')`는 0이고
- `parseInt`는 NaN이 된다.

## JS 숫자는 64비트 부동소수점 (책 캡처 — 출처 불명)

> 대다수의 프로그래밍 언어에는 정수 타입과 부동소수점 타입이 따로 있지만 자바스크립트에는 타입이 없으므로 숫자를 모두 64비트 부동소수점으로 표현합니다. 이는 C나 Java에서 사용하는 double 타입에 해당합니다.
>
> 표현할 수 있는 최댓값: `1.7976931348623157×10^308`
>
> 표현할 수 있는 최솟값: `4.940656458412465×10^-324`
>
> 정수 값은 `-2^53 = -9007199254740992` ~ `2^53 = 9007199254740992` 범위의 값을 정확하게 처리할 수 있습니다. 단, 배열 인덱스와 비트 연산만큼은 32비트 정수로 처리합니다.

| 분류 | 표기법 | 예시 | 설명 | 비고 |
|---|---|---|---|---|
| 정수 리터럴 | 10진수 | `123` | 정수를 그대로 표현 | |
| | 16진수 | `0x2a` | 숫자 앞에 `0x`를 붙임 | |
| | 8진수 | `0o73` | 숫자 앞에 `0o`를 붙임 | ES6 |
| | 2진수 | `0b101` | 숫자 앞에 `0b`를 붙임 | ES6 |
| 부동소수점 리터럴 | 정수.소수 | `3.14` | 소수를 그대로 표현 | |
| | 소수 | `0.123` | 정수부가 0일 때는 0을 생략할 수 있음 | |
| | 가수부e지수부 | `6.02e23` | 6.02×10^23 | |
| | 가수부E지수부 | `1.616199E-35` | 1.616199×10^-35 | |

## toFixed 별로임

생각보다 `toFixed` 별로임. 반올림할 일이 잘 없음. 소수점을 n자리부터 끊으려고 많이 썼는데,

1.
```js
(1234.456789).toFixed(8)
// "1234.45678900"
```

0000으로 채우는 것도 보기 흉하고

2.
반**올림** 이것도 찝찝함. 특히 수수료나 출금예정이나 재산 보여줄 때… 사기치는 거잖아 이거.

그래서 `toFixed` 자체를 모든 코드에서 앞으로 쓰지 않기로 하고, `decimalRoundOff()`를 쓰기로 하자.

그러고 보니 그러면 `decimalSlide`도 이제 필요없으려나 둘 다 목적은 똑같긴 한데 접근방법이 다름.
