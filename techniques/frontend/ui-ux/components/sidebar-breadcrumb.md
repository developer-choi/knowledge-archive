# Answers

## Sidebar, Breadcrumb, URL Hierarchy를 같이 설계할 때 핵심 원칙은 무엇인가?

### User Answer
웹 사이트의 URL Hierarchy, 사이드바 메뉴 Hierarchy, Breadcrumb Hierarchy 이 3개가 완벽하게 일치해야 작업이 편함.

**[Breadcrumb - URL - 사이드바 메뉴] 텍스트 통합**

사이드바 메뉴에 나오는 텍스트와 Breadcrumb에 나오는 링크 텍스트는 서로 같아야 혼선이 없음. 즉, 서로 코드 통일이 필요함.

**Ideal: URL Hierarchy 기준 단일 데이터 소스**

```
/section > 섹션명
  /section/sub-a > 서브A
    /section/sub-a/item-1 > 아이템1
    /section/sub-a/item-2 > 아이템2
  /section/sub-b > 서브B
```

이런 식으로 URL마다 텍스트를 정의해두면:
- 사이드바 액티브 조건: 현재 URL이 `/section`으로 시작하는지 체크
- Breadcrumb: URL을 `/` 기준으로 잘라 각 뎁스 텍스트를 재사용
- 중복 코드 없음

사이드바에 없는 더 깊은 뎁스(예: 리스트 상세페이지 `/section/sub-a/1`)가 있으므로, 사이드바 Hierarchy가 아닌 URL Hierarchy 구조에 맞게 데이터를 하나 만들어두고, 여기서 일부를 사이드바가 쓰고 일부를 Breadcrumb가 쓰게 만들면 됨.

**Bad: Hierarchy 불일치 시 번거로움**

사이드바 메뉴 이름과 URL 구조가 불일치하면 Breadcrumb를 하위 페이지마다 각각 따로 따로 만들어야 함.

**Limitation: 사이드바 메뉴 구성에 의존성이 생김**

URL처럼 사이드바 메뉴도 계층적으로 완벽하게 분리하기 위해 사이드바 메뉴를 더 작게 분리할지 말지 고민을 하게 됨:
- 1뎁스 통합 방식: 사이드바 메뉴마다 액티브 효과 계산 방식을 개발에서 수정해야 함.
- 1뎁스 세분화 방식: 1뎁스 사이드바 메뉴가 엄청나게 많아질 수 있음.

**결론: Breadcrumb vs Sidebar 유지 부담**

1. Breadcrumb 마크업 자체는 매우 쉽다.
2. 하지만 수십개 웹페이지를 계층적으로 잘 분리해서 관리할 수 없다면 누군가 고생한다.
   - 페이지 갯수가 수십개면 Breadcrumb도 수십개 → Breadcrumb에 있는 뎁스 갯수만큼 텍스트 → 페이지 제목·링크 둘 다 수백개씩 된다.
   - 페이지 제목 혹은 URL이 바뀌면 그걸 가리키는 Breadcrumb 수십개를 다 찾아서 수정하거나, 페이지 수십개마다 [제목-URL] 한 셋트를 만들고 재사용하거나.
3. 하지만 사이드바는 페이지마다 하나씩 밖에 없기 때문에 훨씬 덜하다.

## Sidebar의 기본 요구사항은 무엇인가?

### User Answer
- **n뎁스 Nested Collapsible**: 메뉴를 접고 펼칠 수 있어야 함. 펼침여부에 따라 아이콘 방향이 달라지는 경우도 있음.
- **스크롤 대응**: 메뉴를 펼치면 사이드바 높이가 뷰포트 높이보다 길어질 수 있음. 대체로 사이드바 상단에 로고·프로필 정보가 나오는데, 이 영역은 스크롤이 되더라도 고정인 경우가 많음.

## Sidebar의 마크업 요소는 어떻게 결정하는가?

### User Answer
펼칠 수 있으면 `<button>`, 링크면 `<a>`가 맞음. 두 경우 모두 포커스는 받아야 하니까.

## Sidebar 메뉴 아이템의 액티브 효과는 어떻게 설계하는가?

### User Answer
1뎁스, 2뎁스, n뎁스 전부 다 링크일 수도 있고 아코디언일 수도 있음.

- 2뎁스가 **없는** 1뎁스: 펼칠 수 없고 누르면 바로 페이지 이동 → `href` 필요
- 2뎁스가 **있는** 1뎁스: 펼칠 수 있고, 현재 웹페이지가 하위 뎁스에 존재하는지 체크해야 → `href` 필요

`href`가 항상 필요하지만, 링크일 수도 아닐 수도 있어야 함.

## Sidebar 첫 로딩 시 액티브 상태는 어떻게 처리하는가?

### User Answer
1. 접속한 URL에 매칭되는 페이지가 속한 상위 뎁스를 펼치고 활성화 스타일 넣어놓고,
2. 그 상위뎁스 아래에 정확히 일치하는 사이드바 메뉴가 있다면 그 메뉴 액티브 스타일 넣어야 함.
   - 사이드바에 어드민의 모든 페이지가 있지 않으므로 2번이 조건에 따라 갈림.
   - 3차 4차까지 뎁스가 있다면 3차 4차까지 다 찾아서 열려야 하는데, 이 때는 애니메이션이 없는게 자연스럽긴 함.

## 1뎁스 링크 메뉴와 아코디언 메뉴가 공존할 때 생기는 예외 케이스는?

### User Answer
현재 링크 메뉴(예: 홈페이지)에 있을 때 아코디언 1뎁스 메뉴를 클릭하면:

- 해당 아코디언이 펼쳐지지만 페이지 이동은 안 됨.
- 다시 [홈페이지] 링크를 누르면 이미 홈이라 페이지 이동을 안 함.
- 그 대신 펼쳐진 아코디언을 닫기 위해 [홈페이지] 링크를 눌렀을 때에도 아코디언 토글 로직을 실행시켜야 함.

## Breadcrumb 구현 시 주의할 점은 무엇인가?

### User Answer
- 1뎁스 페이지로 들어와서 Breadcrumb도 딱 하나만 노출되야 하는 케이스가 있다면 필요없을거같다.
- A > B > C 이런식으로 보이기 때문에, 구분자는 갯수보다 1개만큼 적다는걸 주의해야 함.
