# [UNVERIFIED] Physical Layer의 역할은 무엇인가?

## 도입

OSI 7계층의 맨 아래, Physical Layer는 데이터를 실제 물리적 신호로 변환해 전선·공기를 통해 전달하는 계층이다. 상위 계층의 0과 1이 여기서 전압 변화·빛·전파로 변환된다.

---

## 본문

Physical Layer는 다음 세 가지 역할을 한다.

1. **물리적 연결**: 장치들을 케이블·와이파이·광섬유로 연결한다.
2. **데이터를 0과 1로 변환**: 상위 계층의 디지털 데이터를 물리 신호(전압, 빛 펄스, 전파)로 인코딩하고, 수신 시 신호를 다시 0과 1로 복원한다.
3. **Transmission media를 통해 전송**: 변환된 신호를 전송 매체(케이블, 광섬유, 공기)를 통해 보낸다.

```
Physical Layer의 동작

상위 계층: 01001000 01100101 ... (디지털 데이터)
                ↓
Physical Layer: 0 → 낮은 전압, 1 → 높은 전압 (또는 빛 펄스)
                ↓
전송 매체: 전선 (이더넷), 광섬유, 공기 (와이파이)
                ↓
수신 측 Physical Layer: 신호 → 0과 1 복원
                ↓
상위 계층: 01001000 01100101 ...
```

---

## 종합

Physical Layer는 소프트웨어 관점에서는 거의 보이지 않지만, 모든 네트워크 통신의 물리적 기반이다. 와이파이 신호가 약하거나 케이블이 낡으면 Physical Layer에서 오류가 발생하고, 이것이 상위 계층의 재전송으로 이어진다. DevTools에서 요청이 느릴 때 Physical Layer 문제일 수도 있다.

# [UNVERIFIED] LAN Card란 무엇인가?

## 도입

LAN Card(랜카드)는 컴퓨터에서 Physical Layer 역할을 담당하는 하드웨어다. 데이터를 물리 신호로 변환하고 복원하는 칩이 여기 들어 있다.

---

## 본문

LAN Card는 Physical Layer 장비다. 내가 보낼 데이터를 0과 1로 바꿔주고, 받은 0과 1 데이터를 복원해주는 역할을 한다.

정확히는 디지털 데이터를 전기 신호(이더넷 케이블) 또는 전파(와이파이)로 변환하고, 수신 시 그 반대 변환을 수행한다. 이 변환을 "인코딩/디코딩"이라고 한다.

데스크탑 기준으로 랜카드는 메인보드에 내장되어 있는 경우가 대부분이다. 노트북도 와이파이 카드가 내장되어 있어 외부에서 보이지 않는다.

```
LAN Card (NIC)의 위치

컴퓨터 내부:
CPU → 메모리 → OS 네트워크 스택 → NIC 드라이버 → [LAN Card]
                                                        ↓
                                               RJ45 포트 / 안테나
                                                        ↓
                                            이더넷 케이블 / 전파
```

---

## 종합

LAN Card(NIC)는 소프트웨어 세계(0과 1)와 물리 세계(전기 신호·전파) 사이의 번역기다. 기가비트 LAN Card는 1Gbps의 데이터를 실시간으로 인코딩·디코딩하는 고성능 하드웨어다.

---

---

# [UNVERIFIED] Network Adaptor와 NIC(Network Interface Controller)의 관계는?

## 도입

랜카드·NIC·Network Adaptor가 혼용되는데, 이 용어들의 관계를 명확히 정리한다.

---

## 본문

용어 관계:

- **Network Adaptor**: 컴퓨터를 네트워크에 연결하는 장치를 통칭하는 넓은 표현이다. 랜카드, USB 동글이, 와이파이 카드, Thunderbolt 어댑터 등이 모두 여기 포함된다.
- **NIC (Network Interface Controller / Network Interface Card)**: Network Adapter와 동의어로 쓰인다. 특히 물리 카드 형태(PCI/PCIe 슬롯)를 강조할 때 쓰는 경향이 있다.
- **LAN Card**: 로컬 네트워크(LAN)에 연결하기 위한 Network Adaptor. 일상 언어에서 NIC와 거의 동의어로 쓰인다.
- **동글이**: USB 포트에 꽂는 Network Adaptor. 내장 랜카드가 없거나 추가 인터페이스가 필요할 때 사용한다.

```
Network Adaptor 종류

Network Adaptor
├── 내장 이더넷 카드 (NIC, 메인보드 내장)
├── PCIe 이더넷 카드 (확장 슬롯 추가)
├── 내장 와이파이 카드
├── USB 이더넷 동글이
└── USB 와이파이 동글이
```

---

## 종합

NIC·Network Adaptor·LAN Card는 맥락에 따라 구분 없이 혼용된다. 공식 문서에서는 NIC(Network Interface Controller)가 표준 용어다. 운영체제에서 네트워크 인터페이스를 `eth0`, `wlan0`, `en0` 같은 이름으로 구분하는 것도 장치마다 별도 NIC가 있기 때문이다.

# [UNVERIFIED] Transmission Media란 무엇이며, 어떤 특성이 있는가?

## 도입

Physical Layer에서 신호를 전달하는 물리적 경로가 Transmission Media(전송 매체)다. 매체의 종류에 따라 최대 전송 속도와 전송 거리가 달라진다.

---

## 본문

Transmission Media는 Physical Layer 장비(개념)다. 신호가 이동하는 물리적 경로 자체를 말한다.

크게 유선과 무선으로 나뉜다:

**유선**:
- 이더넷 케이블 (Cat5e, Cat6, Cat8): 전기 신호. 거리와 카테고리에 따라 100Mbps~40Gbps.
- 광섬유: 빛 펄스. 장거리·고속. 데이터센터 간 연결, 해저 케이블에 사용.
- 동축케이블: 케이블 TV, 일부 인터넷 연결.

**무선**:
- Wi-Fi (2.4GHz, 5GHz, 6GHz): 전파. 장애물에 영향 받음.
- 이동통신 (4G LTE, 5G): 이동 중 사용 가능.

매체마다 속도가 다르다. 실생활 예시: 1기가 인터넷 요금제를 써도 랜케이블이 Cat5e 이하(100Mbps 한계)면 이론 속도가 나오지 않는다. 전송 매체가 병목이 된다.

```
전송 매체별 특성 (대략)

유선 이더넷 Cat5e:   최대 1Gbps, 100m
유선 이더넷 Cat8:    최대 40Gbps, 30m
단일 모드 광섬유:    수Tbps 잠재력, km 단위 장거리
Wi-Fi 802.11ac:     수백Mbps~수Gbps, 실내 수십m
```

---

## 종합

Transmission Media는 네트워크 성능의 물리적 한계를 정한다. 아무리 좋은 스위치와 라우터를 써도 케이블이 구형이면 그 한계 이상의 속도가 나오지 않는다. 인프라 업그레이드(기가 인터넷 등)를 할 때 라우터뿐 아니라 케이블도 함께 교체해야 하는 이유다.

# [UNVERIFIED] Repeater는 어떤 장비이며, 왜 deprecated 되었는가?

## 도입

신호는 케이블을 통해 전달될수록 약해진다. Repeater는 이 약해진 신호를 증폭해 더 먼 거리까지 전달하기 위한 Physical Layer 장비다. 그러나 현재는 거의 사용되지 않는다.

---

## 본문

Repeater는 Physical Layer 장비다.

작동 원리:
1. 멀리 있는 컴퓨터와 통신하고 싶다.
2. 거리가 멀수록 신호 세기가 약해진다(감쇠, attenuation).
3. Repeater가 약해진 신호를 감지하고 증폭(amplify)해 재전송한다(파형이 정상으로 복원됨).
4. 이를 통해 케이블의 최대 거리 한계를 넘어 통신할 수 있다.

deprecated된 이유:
- 1:1 통신밖에 하지 못한다 — 두 개 포트만 있다.
- 현재 다른 네트워크 장비(허브, 스위치, 라우터)가 Repeater 기능을 포함하면서 별도 Repeater의 필요성이 사라졌다.
- 광섬유 사용 확산으로 장거리 감쇠 문제가 줄어들었다.

---

## 종합

Repeater의 역할은 현재 스위치와 라우터가 대신한다. 기가비트 스위치는 포트마다 신호를 재생성해 감쇠 없이 다음 장치로 전달한다. Repeater는 네트워크 장비의 진화 역사에서 첫 단계에 해당하는 과거 기술이다.

# [UNVERIFIED] Hub는 어떤 장비이며, Repeater와 어떤 차이가 있는가?

## 도입

Hub(허브)는 Repeater에서 발전한 Physical Layer 장비다. Repeater가 1:1 연결이라면, Hub는 여러 장치를 연결하는 멀티포트 Repeater라고 볼 수 있다. 그러나 더 스마트한 스위치에 밀려 현재는 거의 쓰이지 않는다.

---

## 본문

Hub는 Physical Layer 장비다.

Repeater와의 차이:

1. **Repeater처럼 신호 증폭이 가능하다**: 약해진 신호를 재생성해 연결된 모든 포트로 내보낸다.
2. **Repeater와 달리 여러 대 컴퓨터와 통신할 수 있다**: 멀티포트 구조로 여러 장치를 하나의 허브에 연결할 수 있다.
3. **여러 대 컴퓨터가 통신하다 보면 충돌이 발생할 수 있다**: 공유 매체에서 동시 전송 시 충돌이 발생한다. 이 내용은 Datalink Layer(CSMA/CD)에서 다룬다.

```
장비 진화

Repeater → Hub → Switch
   1:1      1:N    1:N (지능적 전달)
 신호증폭  신호증폭  MAC 학습
 L1 장비   L1 장비   L2 장비

차이점:
- Hub: 모든 포트에 신호 복사 (브로드캐스트)
- Switch: 목적지 MAC 포트에만 전달 (유니캐스트)
```

---

## 종합

Hub는 Multiple Repeater지만 Repeater보다 더 큰 문제를 가져왔다 — 연결 장치가 많을수록 충돌이 잦아지고 성능이 떨어진다. 현재 유선 LAN에서 Hub는 사실상 완전히 스위치로 대체됐다. Hub가 아직 언급되는 이유는 스위치와의 비교를 통해 스위치의 장점(충돌 없음, 정확한 포트 전달, Full Duplex)을 설명하기 위해서다.
