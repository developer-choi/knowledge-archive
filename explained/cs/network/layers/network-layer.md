# [UNVERIFIED] Network Layer의 역할은 무엇인가?

## 도입

Datalink Layer가 같은 네트워크 안의 한 홉 전달을 담당한다면, Network Layer는 서로 다른 네트워크를 넘어 최종 목적지까지의 경로를 책임진다. `fetch("https://example.com")`을 보낼 때 내 LAN에서 미국 서버까지의 경로를 계산하는 게 Network Layer의 일이다.

---

## 본문

Network Layer는 발신지 대 목적지 전달에 대한 책임을 진다. 두 가지 핵심 역할을 한다.

**1. IP 헤더 지정**
- 패킷에 출발지 IP 주소와 목적지 IP 주소를 붙인다.
- 이 IP 주소가 라우터가 경로를 계산할 때 사용하는 기준이다.

**2. Routing (경로 결정)**
- 수많은 네트워크들 속에서 최적의 경로를 찾는다.
- 라우팅 테이블을 참조해 다음 라우터(홉)를 결정한다.
- 다음 라우터로 전달(forwarding)하면 그 라우터가 또 라우팅 결정을 한다.
- 이 과정을 목적지 네트워크에 도달할 때까지 반복한다.

```
Network Layer의 역할

출발지 PC (서울)
    │
    │ L3: IP 헤더 (src: 192.168.0.10, dst: 93.184.216.34)
    ↓
[공유기] → 라우팅 결정 → ISP 라우터로 포워딩
[ISP 라우터] → 라우팅 결정 → 백본 라우터로 포워딩
[백본 라우터] → 라우팅 결정 → 목적지 네트워크로
...
    ↓
목적지 서버 (93.184.216.34, 뉴욕)
```

각 라우터는 IP 헤더의 목적지 IP를 보고 다음 홉을 결정한다 — 이것이 Network Layer의 동작이다.

---

## 종합

Network Layer는 엔드-투-엔드 경로의 책임자다. Datalink Layer가 "이 라우터에서 다음 라우터까지"의 한 홉을 책임진다면, Network Layer는 "출발지에서 목적지까지 전체 여정"을 책임진다. 라우터가 Network Layer 장치인 이유가 바로 이 역할 때문이다.

# [UNVERIFIED] Network Layer는 언제 필요한 계층인가?

## 도입

같은 네트워크 안의 통신이라면 Datalink Layer만으로 충분하다. Network Layer는 다른 네트워크와 통신할 때 비로소 필요해진다.

---

## 본문

Network Layer는 다른 네트워크와 통신할 때 필요한 계층이다. (= 네트워크 간에 통신할 때 필요한 계층)

Datalink Layer의 한계:
- MAC 주소 기반 전달은 같은 LAN 안에서만 유효하다.
- 라우터를 넘어가면 MAC 주소로는 목적지를 찾을 수 없다.
- ARP 브로드캐스트도 같은 서브넷 안에서만 동작한다.

Network Layer가 해결하는 문제:
- IP 주소는 hierarchical addressing — 네트워크 프리픽스로 "어느 네트워크인지"를 인코딩한다.
- 라우터가 목적지 IP의 프리픽스를 보고 경로를 결정할 수 있다.
- 여러 라우터를 거쳐도 목적지 IP가 바뀌지 않으므로 최종 목적지를 추적할 수 있다.

```
같은 네트워크 vs 다른 네트워크

같은 네트워크 (Datalink Layer만으로 충분):
PC-A (192.168.1.10) → 스위치 → PC-B (192.168.1.20)
MAC 주소 기반 전달, 라우터 불필요

다른 네트워크 (Network Layer 필요):
PC-A (192.168.1.10) → 공유기(라우터) → ISP → 서버 (93.184.216.34)
IP 기반 라우팅, 라우터가 경로 결정
```

---

## 종합

개발자 관점에서 Network Layer의 필요성은 `fetch("https://example.com")`이 로컬 서버가 아닌 외부 서버를 호출할 때다. 로컬호스트(`localhost`)는 Network Layer를 실질적으로 건너뛰어 루프백 인터페이스로 처리되므로 훨씬 빠르다. 외부 API를 호출할 때 `localhost`보다 훨씬 느린 이유가 Network Layer(+ Datalink Layer의 여러 홉)를 실제로 거치기 때문이다.
