# [UNVERIFIED] IP란 무엇인가?

## 도입

인터넷에서 데이터를 보내려면 목적지를 알아야 한다. IP(Internet Protocol)는 인터넷에 연결된 각 장치에 고유한 주소를 부여하는 체계다. 이 주소가 없으면 데이터가 어디로 가야 할지 알 수 없다.

---

## 본문

IP(Internet Protocol Address)는 인터넷 상에서 컴퓨터를 식별하는 숫자 주소다. 현재 널리 쓰이는 IPv4는 32비트로 구성되며, 점으로 구분된 4개의 10진수로 표현한다 (예: `192.168.1.1`).

User Annotation이 짚듯 사람을 특정하는 주민번호, 집을 특정하는 우편 주소, 인터넷 상의 컴퓨터를 특정하는 IP 주소는 모두 동일한 필요성에서 출발한다. 데이터를 보내려면 "어디로"를 알아야 하고, IP가 그 "어디"를 담당한다.

```
택배 시스템                  인터넷 통신
수신자 주소 (집 주소)   ←→   IP 주소 (컴퓨터 주소)
송신자 주소             ←→   출발지 IP
배송 경로               ←→   라우터가 결정하는 패킷 경로
```

IP 주소가 없으면 라우터는 패킷을 어디로 보내야 할지 모른다. 택배 주소가 없으면 배달부가 어디로 가야 할지 모르는 것과 같다.

---

## 종합

IP는 인터넷 통신의 기본 단위다. `fetch('https://api.example.com')`을 호출할 때 브라우저는 먼저 DNS를 통해 `api.example.com`의 IP 주소를 조회하고, 그 IP로 패킷을 보낸다. IP 자체는 "보내기"만 할 뿐 정확한 도착이나 순서는 보장하지 않는다. 신뢰성은 위 계층인 TCP가 담당한다.

---

# [UNVERIFIED] Public IP란 무엇인가?

## 도입

모든 IP 주소가 같은 것이 아니다. 인터넷에서 직접 접근할 수 있는 주소(Public IP)와, 내부 네트워크에서만 쓰이는 주소(Private IP)로 나뉜다.

---

## 본문

Public IP는 인터넷에서 직접 라우팅 가능한 IP 주소다. 전 세계에서 유일하다. ISP(Internet Service Provider, 인터넷 서비스 제공업체 — 예: SK브로드밴드, KT)가 고객에게 할당한다.

우리 집의 인터넷 공유기(라우터)는 ISP로부터 Public IP 하나를 받는다. 집 밖에서 우리 집 네트워크로 들어오려면 이 Public IP 주소로 접근해야 한다.

```
ISP (SK 인터넷)
       │
       │ Public IP: 211.45.67.89 (전국에서 유일)
       ▼
   공유기 (라우터)
   ├── 노트북 (Private IP: 192.168.0.2)
   ├── 핸드폰 (Private IP: 192.168.0.3)
   └── TV    (Private IP: 192.168.0.4)
```

외부에서 `211.45.67.89`로 접근하면 우리 집 공유기에 도달한다. 그 다음 공유기가 내부 어느 장치로 보낼지 NAT(Network Address Translation)를 통해 결정한다.

---

## 종합

서버를 외부에 공개하려면 Public IP(또는 도메인)가 필요하다. AWS EC2 인스턴스를 생성하면 Public IP가 할당되고, 이 IP로 전 세계에서 접근할 수 있다. 로컬에서 `localhost:3000`으로 개발 서버를 띄우면 본인만 접근 가능한 것은, 공유기가 이 포트를 외부에 열어두지 않기 때문이다.

---

---

# [UNVERIFIED] Private IP란 무엇인가?

## 도입

인터넷에서 직접 접근할 수 없는 내부 전용 주소가 Private IP다. 같은 네트워크(LAN — Local Area Network) 내에서만 통신할 때 사용한다.

---

## 본문

Private IP는 특정 예약된 주소 대역을 사용한다. 이 대역은 인터넷 라우터가 라우팅하지 않아 외부에서 직접 접근이 불가능하다.

**Private IP 대역 (RFC 1918):**
- `10.0.0.0` ~ `10.255.255.255` (10.x.x.x)
- `172.16.0.0` ~ `172.31.255.255` (172.16.x.x ~ 172.31.x.x)
- `192.168.0.0` ~ `192.168.255.255` (192.168.x.x)

집 공유기에 연결된 장치들은 모두 `192.168.x.x` 대역의 Private IP를 받는다. 회사 내부 서버들은 같은 대역 내에서 Private IP로 서로 통신한다.

User Annotation이 짚듯:
- **Private IP를 만든 이유**: 내부 장치마다 Public IP를 부여하면 IPv4 주소(약 43억 개)가 금방 고갈된다. Private IP 대역을 분리하여 내부에서는 재사용 가능한 주소를 쓰고, 외부에는 공유기의 Public IP 하나만 노출한다.
- **외부 인터넷 접속**: 핸드폰이 Private IP(`192.168.x.x`)를 가지고 있어도 인터넷에 접속할 수 있다. 공유기가 NAT로 Private IP를 Public IP로 변환하여 내보내기 때문이다.

```
내부 통신 (Private IP):
노트북 (192.168.0.2) ──► 프린터 (192.168.0.10)

외부 인터넷 접속 (NAT):
핸드폰 (192.168.0.3) ──► 공유기 ──► (211.45.67.89로 변환) ──► 인터넷
```

---

## 종합

리눅스에서 `ifconfig`, Windows에서 `ipconfig`로 확인하면 `192.168.x.x` 또는 `172.x.x.x` 대역이 보이는데, 이것이 Private IP다. 로컬 개발 환경에서 같은 와이파이의 동료 PC에서 내 개발 서버에 접근하려면 이 Private IP로 접근한다 — Public IP가 아니라.

---

# [UNVERIFIED] Private IP를 외부에서 접근할 수 있으면 어떤 문제가 생기는가?

## 도입

Private IP가 외부에서 접근 가능하다면 내부 네트워크의 모든 장치가 인터넷에 노출된다. 보안과 프라이버시 측면에서 심각한 문제가 생긴다.

---

## 본문

User Answer의 비유가 직관적이다. 옆집 프린터를 마음대로 쓸 수 있게 되는 것과 같다.

**외부 접근이 가능하다면:**

- 집 안의 프린터, NAS(개인 파일 서버), 스마트 TV, IoT 장치(CCTV 등)가 모두 외부에서 접근 가능해진다.
- 보안 설정이 안 된 장치는 즉시 공격 대상이 된다.
- 내부 네트워크 전체를 통해 다른 장치로 이동(lateral movement)하는 공격이 가능해진다.

**실무 적용 (User Annotation):**

```
로컬 개발 환경변수:
DATABASE_URL=postgresql://211.45.67.89:5432/db  ← Public IP

개발/운영 환경변수 (같은 VPC/네트워크 안):
DATABASE_URL=postgresql://10.0.0.5:5432/db  ← Private IP
```

같은 AWS VPC(Virtual Private Cloud) 안에 있는 EC2(앱 서버)와 RDS(DB 서버)는 Private IP로 통신한다. 외부에서 DB에 직접 접근하지 못하도록 Private IP를 사용하는 것이 보안의 기본이다.

로컬 환경에서는 DB가 다른 네트워크(집 vs 회사 데이터센터)에 있으므로 Public IP를 써야 하지만, 배포 환경에서는 같은 VPC 내 Private IP로 통신하므로 더 빠르고 안전하다.

---

## 종합

Private IP 분리는 성능(Public IP 라우팅 거치지 않음)과 보안(외부에서 직접 접근 불가) 두 가지를 동시에 달성한다. AWS에서 DB를 생성할 때 "Public accessible: No"를 선택하는 것이 이 원칙의 적용이다. 내부 서비스 간 통신에는 Private IP, 외부에 공개되어야 하는 서비스(웹서버, API 게이트웨이)에만 Public IP를 사용한다.

# [UNVERIFIED] IP Address의 클래스(A~C)란 무엇인가?

## 도입

IPv4 주소(32비트)는 네트워크 규모에 따라 A, B, C 클래스로 나뉜다. 클래스는 "네트워크 주소"와 "호스트 주소"를 몇 비트씩 나눠 쓰는지를 결정한다.

---

## 본문

IP 주소는 두 부분으로 나뉜다:
- **Network ID**: 어느 네트워크인지 (우편 주소의 도시/구)
- **Host ID**: 그 네트워크 안의 어느 장치인지 (우편 주소의 번지)

**A 클래스**

앞 8비트가 Network ID, 뒤 24비트가 Host ID다. 하나의 네트워크에 호스트를 약 1600만 개(`2^24 - 2`) 수용할 수 있다. 대형 ISP, 대기업, 정부 기관이 사용한다.

```
형식: N.H.H.H
예:  10.x.x.x (N=10, 나머지는 Host ID)
범위: 0.0.0.0 ~ 127.255.255.255
```

**B 클래스**

앞 16비트가 Network ID, 뒤 16비트가 Host ID다. 하나의 네트워크에 호스트 약 65000개 수용. 중규모 기업/기관.

```
형식: N.N.H.H
예:  172.16.x.x
범위: 128.0.0.0 ~ 191.255.255.255
```

**C 클래스**

앞 24비트가 Network ID, 뒤 8비트가 Host ID다. 하나의 네트워크에 호스트 최대 254개(2^8 - 2). 소규모 네트워크, 가정용 공유기.

```
형식: N.N.N.H
예:  192.168.0.x (Network: 192.168.0, Host: 1~254)
범위: 192.0.0.0 ~ 223.255.255.255
```

```
클래스  Network bits  Host bits  최대 호스트 수
A       8              24         약 1600만
B       16             16         약 65000
C       24             8          254
```

---

## 종합

가정용 공유기가 `192.168.0.x`(C 클래스)를 쓰는 것은 집 안에 있는 장치가 보통 254개를 넘지 않기 때문이다. 오늘날 클래스 기반 분류는 CIDR(Classless Inter-Domain Routing)로 대체됐다. CIDR은 `/24`, `/16` 같은 서브넷 마스크로 더 유연하게 네트워크를 분할한다. AWS VPC 설정에서 `10.0.0.0/16` 같은 CIDR 표기를 보는 것이 이것이다.

# [UNVERIFIED] Network Address란 무엇인가?

## 도입

IP 주소 중에는 개별 장치에 할당할 수 없는 특수 목적 주소가 있다. Network Address가 그 중 하나로, 네트워크 자체를 대표하는 주소다.

---

## 본문

Network Address는 Host ID 부분이 모두 0인 IP 주소다. 네트워크 전체를 나타내는 식별자로, 개별 장치에 할당할 수 없다.

**예시:**

C 클래스 네트워크 `192.168.0.x`에서:
- Network Address: `192.168.0.0` (Host ID = 0)
- 할당 가능한 범위: `192.168.0.1` ~ `192.168.0.254`
- Broadcast Address: `192.168.0.255` (Host ID = 255)

```
192.168.0.0    ← Network Address (장치에 할당 불가)
192.168.0.1    ← 첫 번째 호스트 (보통 공유기)
192.168.0.2    ← 두 번째 호스트
...
192.168.0.254  ← 마지막 호스트
192.168.0.255  ← Broadcast Address (장치에 할당 불가)

실제 사용 가능한 호스트 수 = 256 - 2 = 254
```

Network Address는 라우터가 "이 패킷은 192.168.0.x 네트워크 전체에 대한 것"이라고 판단하는 데 사용한다. 라우팅 테이블에 네트워크를 식별하는 항목으로 등장한다.

---

## 종합

AWS에서 VPC를 만들 때 `10.0.0.0/16`을 지정하면 `10.0.0.0`이 Network Address, `10.0.255.255`가 Broadcast Address로 예약된다. 실제 사용 가능한 IP는 이 두 개를 제외한 범위다. AWS는 추가로 몇 개의 IP를 시스템 용도로 예약하므로 실제 사용 가능한 수는 더 줄어든다.

---

---

# [UNVERIFIED] Broadcast Address란 무엇인가?

## 도입

네트워크 내의 모든 장치에 동시에 데이터를 보내야 할 때가 있다. 하나씩 보내면 비효율적이다. Broadcast Address는 네트워크 전체에 한 번에 전달하는 특수 주소다.

---

## 본문

Broadcast Address는 Host ID 부분이 모두 1(= 255)인 IP 주소다. 이 주소로 보내는 데이터는 해당 네트워크의 모든 장치가 수신한다.

**예시:**

`192.168.0.x` 네트워크에서:
- Broadcast Address: `192.168.0.255`
- 이 주소로 패킷을 보내면 `192.168.0.1` ~ `192.168.0.254`의 모든 장치가 받는다

**실제 사용 예 — DHCP Discovery:**

새 장치가 네트워크에 연결될 때 IP를 받기 위해 DHCP 서버를 찾아야 한다. 아직 IP가 없는 상태이므로 직접 연결을 못 한다. 이때 Broadcast Address를 사용한다:

```
새 장치 (IP 없음)
→ 출발: 0.0.0.0 (아직 IP 없음)
→ 목적지: 255.255.255.255 (제한된 브로드캐스트)
→ 메시지: "DHCP 서버 있으면 대답해줘!"
← DHCP 서버만 응답: "나 여기 있어, IP 줄게"
```

Broadcast Address도 Network Address처럼 개별 장치에 할당할 수 없는 예약 주소다.

**왜 UDP를 쓰는가:**

TCP는 연결을 먼저 확립(3-way handshake)해야 하는데, 불특정 다수에게 동시에 연결을 맺는 것은 불가능하다. Broadcast는 UDP를 사용한다 — 연결 없이 그냥 뿌린다.

---

## 종합

Broadcast는 DHCP(IP 자동 할당), ARP(MAC 주소 조회), NetBIOS 이름 확인 등에서 사용된다. 가정용 공유기가 새 장치(노트북, 핸드폰)에 IP를 자동 할당하는 것은 DHCP + Broadcast가 작동한 결과다. 브로드캐스트는 같은 네트워크(LAN) 내에서만 전달되며, 라우터는 브로드캐스트를 다른 네트워크로 전달하지 않는다.
