# Stack이란 무엇인가?

## 도입

Stack은 접시를 쌓는 것처럼 동작하는 자료구조다 — 마지막에 올린 접시를 먼저 꺼낸다. 이 원리를 LIFO(Last In First Out)라고 한다. JS의 `Array.push()`와 `Array.pop()`이 정확히 이 동작이다.

---

## 본문

> A Stack is a linear data structure that follows a particular order in which the operations are performed.

"스택은 연산이 수행되는 특정 순서를 따르는 선형 자료구조다."

- **linear**: 원소들이 한 줄로 이어진 구조. 트리나 그래프처럼 분기가 없다.
- **particular order**: 아무 순서나 아니다 — LIFO 또는 FILO라는 고정된 규칙이 있다.

> The order may be LIFO(Last In First Out) or FILO(First In Last Out).

"순서는 LIFO(후입선출) 또는 FILO(선입후출)일 수 있다."

- **LIFO**: 마지막에 들어온 것이 먼저 나간다. `push(1)`, `push(2)`, `pop()` → 2가 나온다.
- **FILO**: LIFO의 다른 표현. "First In"이 "Last Out"된다는 관점에서 같은 동작을 다르게 표현한 것이다.

```js
const stack = [];
stack.push(1);   // [1]
stack.push(2);   // [1, 2]
stack.push(3);   // [1, 2, 3]
stack.pop();     // 3 반환 → [1, 2]  ← 마지막에 넣은 게 먼저 나옴
```

LIFO의 특징은 push-pop 쌍을 반복하면 스택이 원래 상태로 돌아온다는 점이다. 1을 넣고 1을 빼면 넣기 전과 동일하다.

크기 종류:
- **Fixed Size Stack**: 배열 기반. 모든 동작이 O(1). 단점은 크기 초과 시 overflow.
- **Dynamic Size Stack**: Linked List 기반 또는 동적 배열. 크기 제한 없음.

---

## 종합

Stack의 핵심은 LIFO 순서 보장이다. push와 pop만 허용하고 중간 접근을 차단함으로써, "가장 최근에 들어온 것부터 처리"하는 패턴을 강제한다. 함수 호출 스택(Call Stack)이 대표적인 예시다 — `function A`가 `function B`를 호출하면 B가 스택 위에 쌓이고, B가 끝나야 A로 돌아온다. 브라우저 개발자 도구의 "Call Stack" 패널이 보여주는 것이 정확히 이 구조다.

# [UNVERIFIED] Stack이 가장 적합한 문제 유형의 특징은?

## 도입

Stack은 모든 자료구조 문제에 쓰이는 건 아니다. 특정 패턴의 문제에서만 Stack이 유효하다. 그 패턴의 공통 키워드는 "왼쪽/오른쪽"과 "가장 가까운"이다.

---

## 본문

배열이나 문자열에서 Stack이 유효한 조건:

1. 기준이 되는 값보다 **왼쪽 또는 오른쪽**에서 어떤 조건을 만족하는 값을 찾아야 한다.
2. 그 값이 기준값과 **가장 가까운** 것을 찾아야 한다.

예시 1 — 괄호 쌍 체크:
`{ [ ( ) ] }` 에서 `)`를 만났을 때, 왼쪽에서 가장 가까운 것이 `(`인지 확인한다. "가장 가까운"이 핵심이다 — `( { [ ) ] }` 처럼 순서가 뒤섞인 경우, 가장 가까운 것과 상관없었다면 잘못 매칭된 경우도 통과될 것이다.

예시 2 — 가장 가까운 작은 수 찾기:
`[0, 1, 6, 2]`에서 `2` 기준 왼쪽에서 `2`보다 작은 값을 찾으면 `0`과 `1` 둘 다 조건을 만족하지만, `2`와 **더 가까운** 건 `1`이다.

```
[0, 1, 6, 2]
         ↑ 기준
왼쪽의 작은 값: 0, 1 → 가장 가까운 건 1 (인덱스 1, 기준과 거리 2)
```

Stack이 "가장 가까운" 조건에 자연스럽게 맞는 이유: Stack은 가장 마지막에 넣은 것(=가장 최근에 본 것=가장 가까운 것)을 먼저 꺼낸다.

---

## 종합

"왼쪽/오른쪽에서 가장 가까운 조건 만족값"이라는 패턴은 브루트 포스로 O(n²)이지만, Stack을 쓰면 O(n)으로 줄일 수 있다. Monotonic Stack이 이 패턴의 대표 기법이다. 괄호 매칭, Next Greater Element, 히스토그램 최대 직사각형 등이 이 범주에 속한다.

# [UNVERIFIED] LIFO가 "가장 가까운" 문제에 적합한 이유는?

## 도입

왼쪽 또는 오른쪽에서 "가장 가까운" 값을 찾으려면, 가장 최근에 지나친 값을 먼저 확인해야 한다. LIFO는 정확히 그 순서다.

---

## 본문

"가장 가까운"을 찾으려면 가장 마지막에 본 것부터 확인해야 한다. FIFO라면 가장 먼저 본 것(가장 먼 것)부터 확인하게 되므로 "가장 가까운" 조건과 맞지 않는다.

예시: `[3, 1, 2, 5]`에서 `5`의 왼쪽에서 가장 가까운 작은 수를 찾을 때:
- 직전에 본 것: `2` → `5`보다 작으므로 답은 `2`
- Stack에는 `[3, 1, 2]`가 있고 pop()하면 `2`가 먼저 나온다 (가장 최근 = 가장 가깝다)

```
순회: 3 → 1 → 2 → 5
Stack 상태:     [3] → [3,1] → [3,1,2]
5를 만남 → pop: 2 → 5보다 작은가? YES → 답: 2
```

LIFO 자체가 "가장 마지막에 넣은 것"을 먼저 꺼내는 동작이므로, "가장 최근에 지나친 = 가장 가까운" 개념과 자연스럽게 일치한다.

---

## 종합

LIFO와 "가장 가까운" 조건의 궁합은 구조적이다 — 순회 중에 가장 최근에 만난 원소가 곧 가장 가까운 원소이고, Stack의 top이 항상 가장 최근 원소를 가리킨다. Queue(FIFO)로 같은 문제를 풀려면 전체를 뒤져야 하는 반면, Stack은 top 하나만 확인하면 된다. 이것이 Stack이 "가장 가까운" 패턴 문제에서 핵심 도구가 되는 이유다.

# [UNVERIFIED] Monotonic Stack 패턴이 이중 반복문임에도 O(n)인 이유는?

## 도입

Monotonic Stack 코드를 보면 외부 for 루프 안에 while 루프가 중첩된다. 이중 루프면 O(n²) 아닌가 하는 의문이 드는데, 실제로는 O(n)이다. 그 이유는 "각 반복문의 최악을 곱하는" 게 아니라 "알고리즘 전체에서 실행되는 총 연산 횟수"를 세야 하기 때문이다.

---

## 본문

주요 연산은 push와 pop 두 가지다. 입력 배열 크기를 n이라 할 때:

**push 총 횟수 분석:**
- for 루프는 n번 실행된다.
- 루프 한 번에 push는 정확히 1번만 실행된다.
- 따라서 알고리즘 전체에서 push 총 횟수 = n.

**pop 총 횟수 분석:**
- pop은 while 루프 안에서 실행된다.
- pop을 하려면 해당 원소가 먼저 push되어 있어야 한다.
- push가 총 n번만 일어나므로, 스택에 들어갈 수 있는 원소는 최대 n개다.
- 따라서 pop도 최대 n번 실행될 수 있다.

```
총 연산 = push 횟수 + pop 횟수 ≤ n + n = 2n = O(n)
```

브루트 포스로 구현했다면 O(n²)이다 — 0번 인덱스에서 왼쪽을 훑고, 1번에서 훑고, n번에서 훑어야 하기 때문이다. Monotonic Stack은 순회하면서 "후보가 될 수 있는 것만 스택에 보관"하므로 전체를 반복해서 뒤질 필요가 없다.

```
브루트 포스:
  for i:
    for j < i:        ← i × (i-1) / 2 번 비교 → O(n²)

Monotonic Stack:
  for i:
    while stack.top 조건:
      stack.pop()     ← 총 pop 횟수 ≤ n
    stack.push(i)     ← 총 push 횟수 = n
```

---

## 종합

이중 루프가 O(n)인 이유는 각 원소가 push 한 번, pop 최대 한 번만 실행되기 때문이다. 어떤 원소도 스택에 두 번 들어갈 수 없으므로, 아무리 while 루프가 여러 번 돌아도 총 pop 횟수가 n을 넘을 수 없다. 이처럼 "전체 연산 총합"으로 복잡도를 분석하는 방식을 상각 분석(Amortized Analysis)이라고 한다.
