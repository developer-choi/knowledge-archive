# [UNVERIFIED] 기억장치란 무엇인가?

## 도입

컴퓨터가 데이터를 처리하려면 어딘가에 데이터를 보관할 공간이 필요하다. 이 보관 공간 전체를 기억장치라 부른다. 기억장치는 보관 기간에 따라 임시 보관과 영구 보관으로 나뉘고, 이것이 주기억장치와 보조기억장치의 구분으로 이어진다.

---

## 본문

기억장치는 데이터를 저장하기 위한 장치의 총칭이다. 저장 목적에 따라 두 가지로 나뉜다:

- **임시 저장**: 전원이 켜진 동안만 데이터를 유지한다. 실행 중인 프로그램의 코드와 변수가 여기에 있다.
- **영구 저장**: 전원이 꺼져도 데이터가 유지된다. 파일, 프로그램 설치본이 여기에 있다.

이 둘이 왜 별도로 존재하는가: 임시 저장(RAM)은 빠르지만 비싸고 휘발성이며, 영구 저장(HDD·SSD)은 느리지만 저렴하고 비휘발성이다. 둘 다 같은 특성으로 만들 수 없기 때문에 용도를 나눠 사용한다.

---

## 종합

`node app.js`를 실행하면 `app.js` 파일(영구 저장 = 디스크)이 메모리(임시 저장 = RAM)로 복사되어 프로세스가 된다. 실행 중 변수에 값을 쓰면 RAM에 기록되고, 프로세스가 끝나면 사라진다. 파일 시스템에 `fs.writeFile()`로 저장해야만 디스크에 남는다. 기억장치 개념 없이는 "왜 저장 버튼을 눌러야 하는가"조차 설명할 수 없다.

---

# [UNVERIFIED] 주기억장치란 무엇인가?

## 도입

주기억장치는 CPU가 지금 이 순간 작업하는 공간이다. 전원이 꺼지면 내용이 사라지고, 보조기억장치보다 빠르지만 비싸다. RAM이 가장 대표적인 주기억장치다.

---

## 본문

주기억장치의 핵심 속성:

- **휘발성(Volatile)**: 전원 공급이 끊기면 저장된 모든 데이터가 사라진다. 편집 중인 문서를 저장하지 않고 컴퓨터가 갑자기 꺼지면 작업 내용이 날아가는 이유다.
- **속도**: 보조기억장치(HDD·SSD)보다 읽기/쓰기가 훨씬 빠르다. CPU가 실제로 연산할 수 있는 속도에 맞추려면 빠른 메모리가 필요하기 때문이다.
- **가격**: 같은 용량 대비 보조기억장치보다 비싸다. 그래서 대용량 데이터는 보조기억장치에 보관한다.

동의어: Main Memory, RAM(Random Access Memory).

**JS 비유:** Node.js 프로세스가 실행되는 동안 선언한 모든 변수(`const`, `let`, 배열, 객체)는 RAM에 있다. `process.exit()`로 종료하면 그것들은 모두 사라진다.

---

## 종합

주기억장치 용량이 부족하면 OS가 RAM 내용을 디스크(페이지 파일)로 내보내는 스와핑이 발생하고 성능이 크게 떨어진다. Windows Task Manager의 "메모리" 탭에서 사용률이 90%를 넘으면 느려지는 것이 이 때문이다. 개발 환경에서 `node_modules`가 크고 빌드 작업이 많으면 RAM 사용량이 급등하는 것도 같은 맥락이다.

---

# [UNVERIFIED] 보조기억장치란 무엇인가?

## 도입

프로그램 파일과 사용자 데이터가 컴퓨터를 꺼도 사라지지 않는 이유는 보조기억장치 덕분이다. 주기억장치가 "지금 실행 중인 것"을 담는다면, 보조기억장치는 "지금 실행하지 않아도 오래 보관해야 할 것"을 담는다.

---

## 본문

보조기억장치의 핵심 속성:

- **비휘발성(Non-volatile)**: 전원이 꺼져도 데이터가 보존된다. 파일, 설치된 프로그램, 데이터베이스가 여기에 있다.
- **속도**: 일반적으로 주기억장치보다 읽기/쓰기가 느리다.
- **가격**: 같은 용량 대비 주기억장치보다 저렴하다. 대용량 저장에 적합하다.

동의어: 2차 기억장치. 영어로는 "Storage"를 선호한다 — "Memory"라고 하면 RAM과 혼동되기 쉽기 때문이다.

종류: HDD, SSD, USB 드라이브, NVMe 등.

```
주기억장치 vs 보조기억장치
┌──────────────────┬────────────┬────────────────┐
│                  │ 주기억장치 │ 보조기억장치   │
├──────────────────┼────────────┼────────────────┤
│ 전원 꺼지면      │ 사라짐     │ 유지됨         │
│ 속도             │ 빠름       │ 느림           │
│ 가격(같은 용량)  │ 비쌈       │ 저렴           │
│ 대표 예          │ RAM        │ SSD, HDD       │
└──────────────────┴────────────┴────────────────┘
```

---

## 종합

`npm install`이 완료된 `node_modules`가 다음 세션에도 남아있는 것은 보조기억장치에 저장되었기 때문이다. 반면 실행 중 V8이 생성한 컴파일 캐시는 RAM에 있어서 프로세스가 끝나면 사라진다. 보조기억장치가 없으면 컴퓨터를 켤 때마다 아무 프로그램도 없는 상태에서 시작해야 한다.

# [UNVERIFIED] 기억장치의 가격, 용량, 속도는 어떤 관계인가?

## 도입

기억장치를 단 하나의 종류로 만들 수 없는 이유가 있다. 빠른 메모리는 비싸고 작으며, 저렴한 메모리는 느리고 크다. 이 제약이 계층 구조를 만든다.

---

## 본문

CPU는 보조기억장치에서 데이터를 직접 읽을 수 없다. 반드시 보조기억장치 → 주기억장치(RAM) → (캐시) → CPU 순서로 데이터가 이동해야 한다.

RAM 용량이 작으면 CPU가 필요한 데이터가 RAM에 없는 경우가 자주 발생하고, 그때마다 보조기억장치에서 RAM으로 데이터를 가져와야 한다. 이 과정이 수천 배 느리다. 그렇다고 RAM을 무한정 키우면 비용이 폭증한다.

```
기억장치 계층별 특성
계층       속도    용량     가격(용량당)
─────────────────────────────────────
레지스터   매우빠름  수십 B   ─ (CPU내부)
캐시메모리 빠름      수십 MB  매우비쌈
RAM       보통      수십 GB  비쌈
SSD       느림      수 TB    저렴
HDD       매우느림  수 TB    매우저렴
```

이 트레이드오프가 기억장치를 용도별로 계층화하는 이유다. CPU 접근 빈도가 높은 데이터는 위 계층에, 자주 쓰지 않는 데이터는 아래 계층에 둔다.

---

## 종합

Node.js에서 `import`된 모듈 코드는 처음 로드될 때 디스크(보조기억장치)에서 RAM으로 올라오고, 이후에는 RAM에서 직접 실행된다. `require('fs')`를 1000번 호출해도 디스크 I/O가 1000번 발생하지 않는 이유가 여기 있다. 기억장치 계층 구조는 "비용을 최소화하면서 성능을 최대화"하는 공학적 해결책이다.

# [UNVERIFIED] RAM이란 무엇인가?

## 도입

주기억장치 중 가장 일반적으로 사용하는 것이 RAM이다. RAM의 핵심 특성은 이름에 담겨 있다 — "Random Access", 즉 어느 위치에 있는 데이터든 같은 시간에 접근할 수 있다.

---

## 본문

RAM(Random Access Memory)의 특성:

- **랜덤 접근(Random Access)**: CPU가 주소를 보내면, 그 주소가 메모리 어디에 있든 접근 시간이 동일하다. HDD처럼 헤드가 물리적으로 이동할 필요가 없다. 배열의 `arr[0]`과 `arr[9999]`를 읽는 속도가 동일한 것과 같다.
- **읽기/쓰기 가능**: ROM과 달리 읽기와 쓰기가 모두 가능하다. 실행 중인 프로그램이 변수에 값을 저장하는 것이 여기서 이루어진다.
- **휘발성**: 전원이 끊기면 내용이 사라진다.
- **CPU 주소 기반 접근**: CPU가 주소 버스로 접근할 주소를 보내면 해당 위치의 데이터를 데이터 버스로 주고받는다.

---

## 종합

`const arr = new Array(1000000).fill(0)`를 선언하면 RAM에 약 8MB(64비트 정수 기준)가 할당된다. 이후 `arr[0]`이든 `arr[999999]`든 접근 시간이 동일한 것이 랜덤 접근의 실질적인 의미다. 데이터베이스에서 인덱스로 특정 레코드에 바로 접근할 수 있는 것도 결국 RAM의 랜덤 접근 특성을 활용하는 것이다.

---

# [UNVERIFIED] ROM이란 무엇이며, 주로 어떤 내용이 저장되는가?

## 도입

RAM이 읽고 쓰기가 가능한 휘발성 메모리라면, ROM은 기본적으로 읽기만 가능한 비휘발성 메모리다. 컴퓨터가 처음 켜질 때 필요한 초기화 프로그램처럼, "변하지 않고 항상 있어야 하는 내용"을 담는다.

---

## 본문

ROM(Read Only Memory)의 특성:

- **읽기 전용(Read Only)**: 이름 그대로 읽기만 가능하다. (현대의 일부 ROM은 특수한 방법으로 쓰기도 가능하지만, 기본 개념은 읽기 전용이다.)
- **비휘발성(Non-volatile)**: 전원이 꺼져도 내용이 유지된다. 그래서 컴퓨터 부팅 시 필요한 코드를 저장하기에 적합하다.

주로 저장되는 내용:

- **시스템 초기화 프로그램·진단 프로그램(BIOS/UEFI)**: 컴퓨터를 켰을 때 가장 먼저 실행되는 코드. RAM이 아직 준비되지 않은 상태에서도 읽을 수 있어야 하므로 ROM에 있다.
- **자주 사용되는 함수들과 서브루틴들**: 하드웨어 제조사가 미리 구워둔 기본 루틴.
- **CU(Control Unit)의 마이크로 프로그램**: CPU 내부에서 명령어를 실행하는 방법을 담은 저수준 프로그램.

---

## 종합

컴퓨터의 전원 버튼을 누르면 CPU가 가장 먼저 실행하는 것이 ROM에 있는 BIOS/UEFI 코드다. 이 시점에는 OS도 없고 RAM에도 아무것도 없다. ROM이 없으면 컴퓨터는 켜지는 순간 "다음에 뭘 해야 하는지"를 알 수 없어서 부팅 자체가 불가능하다. RAM이 프로그램의 "작업 공간"이라면, ROM은 프로그램이 시작하기 위한 "출발점"이다.
