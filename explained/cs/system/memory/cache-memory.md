# [UNVERIFIED] 캐시메모리란 무엇인가?

## 도입

CPU는 계산을 매우 빠르게 처리하지만, RAM에서 데이터를 가져오는 속도는 CPU에 비해 훨씬 느리다. 이 속도 차이가 CPU가 데이터를 기다리며 노는 시간을 만들고, 이것이 성능 병목이 된다. 캐시메모리는 이 간격을 메우기 위해 CPU 가까이에 위치한 고속 소용량 메모리다.

---

## 본문

캐시메모리는 CPU와 주기억장치(RAM) 사이에 위치하며, RAM보다 훨씬 빠른 접근 시간(access time)을 제공한다. CPU가 데이터를 읽을 때는 먼저 캐시메모리에 해당 데이터가 있는지 확인한다.

동작 원리:

- 요청한 데이터가 캐시에 있으면 캐시에서 즉시 꺼낸다.
- 없으면 RAM까지 가서 읽어오고, 가져온 김에 캐시에도 복사해 둔다.

```
CPU
 │
 ├─→ 캐시메모리 (L1/L2/L3) ← 먼저 여기서 찾는다
 │        │ 없으면
 └─→ RAM(주기억장치) ← 여기서 가져옴, 캐시에도 복사
```

캐시 용량이 클수록 히트율이 올라가지만, 캐시 자체가 비싸기 때문에 무한정 키울 수 없다. 그래서 크기와 비용 사이의 절충점을 찾는 것이 캐시 설계의 핵심이다.

---

## 종합

V8 엔진이 자주 실행되는 코드 조각을 JIT 컴파일해 두는 것처럼, 캐시메모리도 "자주 쓰는 데이터를 가까이"라는 원칙을 하드웨어 수준에서 구현한 것이다. 캐시메모리가 없으면 CPU는 매번 RAM까지 왕복해야 하고, 이 지연이 누적되면 프로그램 전체 실행 시간이 크게 느려진다. Windows Task Manager의 CPU 사용률이 높지 않은데도 프로그램이 느리다면, 메모리 접근 패턴이 캐시를 제대로 활용하지 못하는 것일 수 있다.

# [UNVERIFIED] Cache Hit와 Cache Miss란?

## 도입

캐시메모리의 효과는 CPU가 원하는 데이터를 캐시에서 찾을 수 있느냐에 달려 있다. 찾을 수 있으면 빠르고, 없으면 RAM까지 가야 한다. 이 두 상황을 각각 Cache Hit와 Cache Miss라 부른다.

---

## 본문

- **Cache Hit (캐시 적중)**: CPU가 요청한 데이터가 캐시메모리에 있는 상태. 빠른 캐시에서 바로 꺼낼 수 있다.
- **Cache Miss (캐시 미스)**: CPU가 요청한 데이터가 캐시메모리에 없는 상태. RAM까지 접근해야 하므로 수십~수백 배 더 오래 걸린다.

```
CPU 데이터 요청
       ↓
  캐시에 있다? ──YES──→ Cache Hit → 즉시 전달 (빠름)
       │
      NO
       ↓
  Cache Miss → RAM 접근 → 데이터 가져옴 + 캐시에 복사
```

캐시 히트율(Hit Rate) = 전체 접근 중 캐시에서 해결된 비율. 히트율이 높을수록 프로그램이 빠르다.

---

## 종합

브라우저의 HTTP 캐시도 같은 원리다. 이미지 파일이 브라우저 캐시에 있으면(Cache Hit) 네트워크 요청 없이 바로 화면에 표시하고, 없으면(Cache Miss) 서버에서 다시 내려받는다. 하드웨어 캐시메모리는 이것을 CPU와 RAM 사이에서 나노초 단위로 반복하는 것이다. 캐시 미스가 많으면 CPU가 데이터를 기다리며 스톨(stall) 상태에 들어가 성능이 급격히 떨어진다.

# [UNVERIFIED] 캐시 설계의 목표는?

## 도입

캐시메모리를 무조건 크게 만들면 히트율은 오르지만 비용도 급등하고, 캐시 자체를 읽는 데 드는 시간도 늘어날 수 있다. 캐시 설계는 세 가지 목표 사이의 균형 문제다.

---

## 본문

캐시 설계의 세 가지 목표:

**1. 캐시 적중률의 극대화**

CPU가 원하는 정보가 캐시에 있을 확률을 높이는 것. 어떤 데이터를 캐시에 올려둘지(교체 정책), 얼마나 큰 범위를 미리 가져올지(prefetch)가 여기에 관련된다.

**2. 캐시 액세스 시간의 최소화**

캐시 적중 시 캐시에서 CPU로 데이터를 전달하는 시간을 단축하는 것. 캐시가 CPU 내부(L1)에 가까울수록 빠르지만 용량이 줄어든다.

**3. 캐시 실패에 따른 지연 시간의 최소화**

캐시 미스가 발생했을 때 RAM에서 읽어오는 시간을 최소화하는 것. 프리페치(prefetch)나 캐시 라인 단위 전송이 여기에 관련된다.

```
캐시 계층 (위로 갈수록 빠르고 작음)
L1 캐시 ─ CPU 칩 내부, 수십 KB, ~1ns
L2 캐시 ─ CPU 칩 내부, 수백 KB, ~5ns
L3 캐시 ─ CPU 칩 내부(공유), 수 MB, ~20ns
RAM     ─ 외부, GB 단위, ~60~100ns
```

---

## 종합

세 목표는 서로 충돌한다. 히트율을 높이려면 캐시를 키워야 하지만, 캐시가 커지면 액세스 시간이 늘어난다. 캐시 미스 지연을 줄이려면 RAM을 빠르게 해야 하지만 비용이 증가한다. 실제 CPU 설계는 L1/L2/L3 다중 계층으로 이 세 목표를 계층별로 분산시켜 균형을 잡는다.
