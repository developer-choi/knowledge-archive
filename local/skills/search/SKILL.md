# Search

## 목적

정보를 언제나 신뢰할 수 있는 1차 출처에서만 탐색한다.

## 탐색 순서

아래 순위를 엄격히 따른다. 상위 출처에서 결과를 찾으면 즉시 멈추고 하위 출처로 내려가지 않는다. KA 내부에서 찾았으면 외부 검색을 시작조차 하지 않는다.

| 순위 | 출처 | 적용 도메인 |
|------|------|------------|
| 1 | KA 내부 (`knowledge/`) | 항상 먼저 — 이미 정리된 Q&A가 있으면 외부 탐색 생략 |
| 2 | 라이브러리·프레임워크 공식문서 | 질문 대상이 특정 기술인 경우 (React, Next.js, TanStack Query 등) |
| 3 | 도메인별 공식 1차 출처 | Wikipedia가 기본값. 도메인에 따라 우선 출처가 다름: AI/LLM 원론 → Wikipedia, Claude·Anthropic API → docs.anthropic.com, 웹 표준·브라우저 → MDN, OS 개념 → OSTEP |
| 4 | 공신력 있는 공개 출처 | 3순위에서 찾지 못한 경우에만. 표준 기관·학술 기관·공식 스펙 문서, 스택오버플로우(커뮤니티 검증 답변), 네임드 개발자 개인 블로그 허용. 출처 불분명한 일반 블로그는 사용하지 않는다 |

## 탐색 규칙

- 공식문서 원문(영어)을 Official Answer로 사용한다
- 탐색 결과를 요약·합성하지 않는다 — 원문을 그대로 인용한다
- 1차 출처에서 찾지 못한 경우, 사용자에게 "찾지 못했다"고 보고하고 대안 출처를 제안한다. 블로그·스택오버플로우 등 2차 출처로 채우지 않는다

## 보고 양식

필수 필드:
- **출처 URL**: 탐색한 페이지의 정확한 URL
- **원문 인용**: 근거가 된 원문 발췌 (영어)
- **한줄 요약**: 원문을 기반으로 한 한 줄 해설 (선택 — 사용자가 개념 이해를 요청한 경우)

## digest 연계

외부 출처(2순위 이하)에서 결과를 찾은 경우, 보고 마지막에 한 줄 제안을 붙인다:

> `/digest ON [출처 URL]` 로 이 문서를 같이 읽으며 Q&A로 저장할 수 있어요. 할까요?

KA 내부에서 찾은 경우에는 제안하지 않는다.
